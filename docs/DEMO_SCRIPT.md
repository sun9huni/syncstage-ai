# SyncStage AI - 피칭 대본 (3분 데모)

> **🎤 발표자 준비사항:** 
> - 라이브 URL: `https://syncstage-ai.vercel.app`
> - 화면 최대화 (전체화면 F11 권장)
> - 오디오 출력 볼륨 적정 수준 확인
> - 비상시: `⚡ LOAD DEMO PRESET` + Quick Patch 버튼으로 API 없이 진행 가능

---

## ⏱️ 0:00 - 인트로 (Problem & Solution)
"안녕하세요, 대한민국 엔터테인먼트 산업의 고질적인 병폐를 해결할 혁신, **SyncStage AI** 팀입니다.
완벽한 K-pop 무대를 만들기 위해 A&R 디렉터와 안무가, 비주얼 팀이 매번 수십 번씩 피드백을 주고받는 과정을 아시나요? 
음원 하이라이트 구간에 어떤 안무가 들어가야 할지, 어떤 의상이 어울릴지 소통하는 물리적, 금전적 비용은 막대합니다.

저희는 **Gemini 2.0 Flash와 Function Calling 기술을 활용해, 음원을 넣기만 하면 즉각적으로 초안을 짜고 자연어로 수정할 수 있는 AI A&R 디렉터 에이전트**를 만들었습니다. 한마디로 K-Pop 시각화의 프리텐더(Pre-tender)입니다."

## ⏱️ 0:30 - 데모 1: 음원 자동 분석 & 초안 (The Magic)
"(헤더 우측의 [⚡ LOAD DEMO PRESET] 버튼 클릭)
자, 기획사에 새로운 데모 음원이 도착했습니다. 버튼 하나로 음원과 AI 분석 결과가 즉시 로드됩니다.

(재생 버튼 ▶ 클릭)
음악이 흘러나오면서 **좌측의 3D 아바타**를 보세요.
Mixamo 스켈레톤 기반의 댄서가 타임라인에 맞춰 **실제 안무 모션으로 전환**됩니다.
인트로의 가벼운 idle → 6.5초에 비트 드롭과 함께 arms_hiphop 파워 무브!

(우측 중앙 'Choreo Timeline' 패널 지목)
5개 세그먼트가 시간순으로 배치돼 있고, 현재 재생 구간이 하이라이트 됩니다."

## ⏱️ 1:20 - 데모 2: 자연어 패치 & 리비전 (Function Calling)
"기존 AI 도구들의 가장 큰 한계는 **'부분 수정의 어려움'**이었습니다. 
저희의 핵심 경쟁력은 **'의도(Intent) 기반 패치'**입니다.

(채팅창에 입력: '도입부 안무를 힙합 그루브로 바꾸고 분위기를 더 어둡게 해줘')
이처럼 A&R 디렉터가 피드백을 주면, Gemini는 전체 데이터를 새로 쓰는 것이 아니라 **Function Calling**을 통해 `update_segment`와 `update_style` 도구를 정확히 골라 **핵심 구간만 타겟하여 수정**합니다. 

(화면 헤더 Revision 지목)
보이시나요? **Revision**이 '1'로 올라가며 데이터 무결성을 증명합니다.
동시에 3D 아바타의 모션과 스포트라이트 색상이 즉각 변경됩니다."

> **💡 [PLAN B - API 지연 시]**
> *(에러 토스트가 뜨면, 우측 하단 Quick Patch 버튼 클릭)*
> *"생방송 데모라 현재 트래픽이 몰리고 있네요! 하지만 보시다시피 저희는 서버 장애에도 멈추지 않는 **오프라인 엣지 폴백**을 구현해 두었습니다. 이것이 바로 프로덕션 레디 아키텍처입니다."*

## ⏱️ 2:10 - 데모 3: 비주얼 콘셉트 시각화 (Imagen 3)
"안무가 정해졌으니, 무대 의상을 볼까요?
(좌측 '👗 Stage Wardrobe' 패널의 [Visualize Concept] 버튼 클릭)

스켈레톤 로딩 애니메이션 → 이미지 생성!
Gemini가 도출한 'Cyberpunk Streetwear' 프롬프트를 **Imagen 3**이 3:4 세로 비율의 K-pop 매거진 퀄리티 이미지로 렌더링합니다.

(우측 상단 배지 지목)
'✦ Imagen 3' 배지 — 구글의 최신 이미지 생성 모델이 만든 결과물입니다."

## ⏱️ 2:40 - 아웃트로 (Vision)
"SyncStage AI는 단순한 안무 생성 장난감이 아닙니다. 
현업 기획자, 안무가, 스타일리스트가 하나의 시각화된 대시보드를 보며 자연어로 소통하게 만드는 **차세대 엔터테인먼트 B2B 협업 툴(Collaboration Tool)**입니다. 무작위 비디오 생성이 아닌, JSON 기반의 '데이터 오케스트레이션'을 택했기에 상용화가 가능한 아키텍처입니다.

오직 Gemini 2.0의 **네이티브 오디오 딥리스닝**, 상태를 통제하는 **Function Calling 에이전트**, 그리고 **Imagen 3 최고 품질 비주얼 모델**의 삼위일체가 있었기에 이 속도와 안정성이 가능했습니다. 음악을 눈으로 보게 만드는 싱크스테이지였습니다. 감사합니다!"
